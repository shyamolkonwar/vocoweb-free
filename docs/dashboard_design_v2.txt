## 1. The Core Layout Strategy

**Problem:** The current "Top Navigation" is cluttered and pushes content down.
**Solution:** A **Fixed Sidebar Layout**.

* **Left (Sidebar):** Permanent navigation. Contains Logo, Primary Links (Websites, Leads), and User Profile.
* **Right (Main Content):** The scrollable area where your `Suspense` components live.
* **Top (Header):** Minimalist header for "Page Title" (e.g., "My Websites") and Global Actions (e.g., "Language Switcher").

---

## 2. Visual Specification (Tailwind CSS)

### Color Palette

* **Sidebar Background:** `bg-slate-900` (Dark Mode) OR `bg-white` with a subtle right border (`border-r border-slate-200`). *Recommendation: Go with White for a clean, clinical look.*
* **Active Link:** `bg-teal-50 text-teal-700 border-r-2 border-teal-600`.
* **Inactive Link:** `text-slate-500 hover:bg-slate-50 hover:text-slate-900`.
* **Typography:** Use `Inter` or your system font. Headings should be `font-semibold text-slate-800`.

### Iconography

Use **Lucide React** (standard for Next.js).

* Websites: `<LayoutGrid />`
* Leads: `<Users />`
* Popups: `<MessageSquarePlus />`
* Settings: `<Settings />`

### Step B: The Sidebar Component

**File:** `src/app/dashboard/components/Sidebar.tsx`

```tsx

  { name: "Billing & Credits", href: "/dashboard/credits", icon: CreditCard },

export default function Sidebar() {

  return (

      <nav className="flex-1 py-6 px-3 space-y-1">
        {navItems.map((item) => {

          const Icon = item.icon;
          
          return (
            <Link
              key={item.href}
              href={item.href}

                flex items-center gap-3 px-3 py-2.5 rounded-lg text-sm font-medium transition-all

              `}
            >
              <Icon size={18} className={isActive ? "text-teal-600" : "text-slate-400"} />
              {item.name}
            </Link>
          );
        })}
      </nav>

      {/* Bottom Profile Area */}
      <div className="p-4 border-t border-slate-100">
         {/* Insert your User Avatar Component here */}
         <div className="flex items-center gap-3">
            <div className="w-8 h-8 rounded-full bg-slate-200"></div>
            <div className="text-sm">
                <p className="font-medium text-slate-700">Shyamol K.</p>
```

ðŸ“± Mobile Responsiveness Strategy (Laxizen Dashboard)
1. Executive Summary
The desktop dashboard relies on a Fixed Sidebar for navigation. On mobile screens (under 768px), this sidebar takes up too much space and must be hidden.

The Solution: We will implement a "Slide-Over Drawer" navigation system.
Desktop: Sidebar is always visible (Fixed).
Mobile: Sidebar is hidden by default. A "Hamburger Menu" button in the header opens it as an overlay.
2. Navigation Architecture
A. The Hamburger Menu Trigger
Location: Add a "Menu" icon button to the far left of the Top Header.
Visibility Rule: This button should only appear on mobile screens. On desktop, it must be hidden since the sidebar is already visible.
Interaction: Tapping this button triggers the Sidebar to slide in from the left.
B. The Mobile Sidebar (Slide-Over)
Instead of creating a separate "Mobile Menu," we will reuse the existing Sidebar component but wrap it in a Sheet / Drawer container.
Behavior:
Closed State: Hidden off-screen (translated -100% to the left).
Open State: Slides in to cover 80% of the screen width.
Overlay: A semi-transparent black background (bg-black/50) must appear behind the sidebar to dim the main content.
Closing Logic: Tapping the dark overlay or a "Close (X)" button inside the sidebar must close it.
Auto-Close: When a user clicks a link (e.g., goes to "Leads"), the mobile sidebar must automatically close.
3. Layout Adaptations (Grid & Spacing)
The main content area currently uses a rigid grid. This must become fluid.

A. The "Stacking" Strategy
Dashboard Cards:
Desktop: The "My Websites" and "Credits" cards sit side-by-side (grid-cols-2).
Mobile: These must stack vertically (grid-cols-1). Each card should take up 100% of the width.
Lead Lists:
If you display leads in a table rows, switch to a "Card View" on mobile. Instead of a wide row with many columns, show a vertical stack of info for each lead.
B. Padding & Margins
Desktop screens have luxury space (32px or p-8). Mobile screens do not.
Container Padding: Reduce the main page padding from p-8 (Desktop) to p-4 (Mobile).
Card Internal Padding: Reduce padding inside cards from p-6 to p-4.
Gap Adjustment: Reduce the gap between grid items from gap-6 to gap-4.
4. Header Transformations
The Top Header needs to be decluttered for small screens.
Logo: If the Logo is normally in the Sidebar, it might be invisible on mobile (since the sidebar is hidden). You may need to display a small Laxizen Logo in the center of the Mobile Header.
User Profile: Keep the User Avatar on the right side of the header. It is the primary way to access settings/logout.
Secondary Actions: Hide non-essential buttons (like "Language Switcher") inside the Mobile Sidebar or the User Dropdown to save header space.
5. Touch Interaction Standards
Mobile users use fingers, not mouse pointers.
Touch Targets: Ensure all buttons and links (especially in the Sidebar) are at least 44px height. Increase vertical padding on sidebar links if necessary.
Active States: Remove "Hover" effects on mobile (since hover doesn't exist). Focus on "Active" states (highlighting the current tab) so the user knows where they are.
Input Fields: Ensure form inputs (like "Search Leads") have a font size of at least 16px. If smaller, iPhones will zoom in automatically, which breaks the UI
